# Styling

The message component provides the following CSS classes for customization:

## Available Classes

### Base Classes
- **`app-message`** - Default style class for the message component
- **`app-message-rtl`** - Applies right-to-left text direction styling for RTL languages

### Dark Theme Message Types
- **`success-dark-message`** - Dark theme styling for success messages with appropriate green color scheme
- **`error-dark-message`** - Dark theme styling for error messages with appropriate red color scheme
- **`warning-dark-message`** - Dark theme styling for warning messages with appropriate orange/yellow color scheme
- **`info-dark-message`** - Dark theme styling for informational messages with appropriate blue color scheme
- **`loading-dark-message`** - Dark theme styling for loading messages with appropriate neutral color scheme

### Light Theme Message Types
- **`success-light-message`** - Light theme styling for success messages with appropriate green color scheme
- **`error-light-message`** - Light theme styling for error messages with appropriate red color scheme
- **`warning-light-message`** - Light theme styling for warning messages with appropriate orange/yellow color scheme
- **`info-light-message`** - Light theme styling for informational messages with appropriate blue color scheme
- **`loading-light-message`** - Light theme styling for loading messages with appropriate neutral color scheme

These classes allow you to customize the appearance of messages based on their type (success, error, warning, info, loading) and theme preference (dark or light), while maintaining consistent visual hierarchy and accessibility standards.
