# Overview

The **Message** component provides a standardized way to display contextual feedback messages to users in mobile applications. It supports different message types including success, error, warning, info, and loading states, each with appropriate visual styling and iconography.

The component offers flexible display options with customizable animations, accessibility support, and programmatic control for showing and hiding messages. It's particularly useful for displaying form validation results, API response feedback, or general user notifications.

## Features

- **Multiple message types**: Success, error, warning, info, and loading states
- **Customizable appearance**: Light and dark variants with type-specific styling
- **Animation support**: Configurable entrance animations with delay options
- **Accessibility compliant**: Built-in screen reader support and ARIA roles
- **Programmatic control**: Methods to show and hide messages dynamically
- **Flexible content**: Support for custom titles and captions
- **Responsive design**: Device-specific visibility controls
- **Optional close button**: Configurable close functionality with custom icons
