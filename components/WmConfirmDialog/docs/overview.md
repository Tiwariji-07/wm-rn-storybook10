# Overview

The ConfirmDialog component is a modal dialog widget designed to prompt users for confirmation before executing critical actions in mobile applications. It provides a standardized way to present important decisions to users, ensuring they acknowledge potentially destructive or irreversible operations such as deleting data, logging out, or making purchases.

This component offers a highly customizable confirmation interface that can be tailored to match your application's design language and user experience requirements. Built with mobile-first principles, it ensures optimal usability across different screen sizes and touch interfaces while maintaining accessibility standards.

## Features

- **Customizable messaging** - Configure title and message content to clearly communicate the action requiring confirmation
- **Flexible button text** - Customize OK and cancel button labels to match the specific context and action
- **Icon support** - Add visual context with customizable icon classes for better user comprehension
- **Smooth animations** - Built-in animation support with configurable delay for polished user interactions
- **Modal behavior** - Optional modal presentation to focus user attention on the confirmation decision
- **Dismissible options** - Configurable close behavior to control how users can dismiss the dialog
- **Event handling** - Comprehensive callback system including onOpened events for tracking dialog lifecycle
- **Mobile-optimized** - Touch-friendly interface designed specifically for mobile user interactions
- **Accessibility compliant** - Built with proper ARIA attributes and keyboard navigation support
