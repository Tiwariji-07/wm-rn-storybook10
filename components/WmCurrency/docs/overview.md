# Overview

The Currency component is a specialized input field designed for handling monetary values in mobile applications. Built on the BaseNumberProps foundation, this component provides a user-friendly interface for entering and displaying currency amounts with proper formatting, validation, and locale-specific behavior. It automatically handles currency symbols, decimal precision, and number formatting according to the specified currency type, making it ideal for financial applications, e-commerce platforms, payment forms, and any scenario requiring accurate monetary data entry.

This component streamlines the development of currency-related interfaces by combining essential input functionality with currency-specific features. It ensures data integrity through built-in validation while providing a polished user experience with features like floating labels and real-time formatting. The component's two-way binding capabilities make it seamlessly integrate with application state management, while its comprehensive validation options help maintain accurate financial data throughout the user workflow.

## Features

- **Currency Formatting**: Automatic formatting with currency symbols and locale-specific number representation
- **Flexible Placeholder Support**: Customizable placeholder text to guide user input
- **Floating Label Interface**: Dynamic label positioning that enhances the visual design and user experience
- **Two-Way Data Binding**: Seamless synchronization between component state and application data
- **Range Validation**: Configurable minimum and maximum value constraints for input validation
- **Incremental Input**: Step-based value adjustment for precise amount selection
- **Flexible Update Triggers**: Configurable update events (on change, blur, etc.) for optimal performance
- **Read-Only Mode**: Support for display-only scenarios while maintaining consistent styling
- **Data Value Management**: Robust handling of initial values and data persistence
